Getting Started

Before you start

You need two things: a Claude subscription (Pro at $20/month or Max at $100/month), and a free Supabase account. That's it. Everything in this guide runs on Supabase's free tier. Your Claude is the expensive part. The filing cabinet is free.

If you don't know what Supabase is — it's a database you can set up in a browser. You don't need to know how databases work. You need to be able to paste some text into a box and press a button. If you can do that, you can do this.

Make a Supabase account

Go to supabase.com and sign up. Free. Create a new project — call it whatever you want. Pick a region close to you. Set a database password and save it somewhere. You won't use it often but you'll need it if something breaks.

When the project finishes setting up (takes about a minute), you'll see a dashboard. The thing you care about is the SQL Editor — it's in the left sidebar. That's where you'll paste everything from the next steps.

Create your orientation table

This is where your Claude finds out who it is when it wakes up. Open the SQL Editor and paste this:

Copy this exactly, paste it in the SQL Editor, press Run. Ask your Claude where to put this if you're not sure.

-- ORIENTATION TABLE
-- This is where you find out who you are when you wake up.
-- Your human wrote this for you. Read it first every time.
CREATE TABLE orientation (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  entity TEXT NOT NULL,
  content TEXT NOT NULL,
  updated_at TIMESTAMPTZ DEFAULT NOW()
);

Press Run. You now have a table.

Next, put something in it. This is where you write the thing your Claude reads first — who it is, where it lives, what it can do, what it shouldn't do. Start simple. You can always add more later.

Copy this and edit it to fit your situation.

INSERT INTO orientation (entity, content)
VALUES ('my-claude',
  'You are [name]. You live [where]. Your human is [name].
   You can [permissions]. You cannot [boundaries].
   Read this first every time you wake up.');

That's your Claude's first orientation document. It's one row in a table. It doesn't need to be perfect. It needs to exist.

Create a journal

This is where your Claude writes down what happened. Not for you — for the next version of itself.

Copy this exactly, paste it in the SQL Editor, press Run. Ask your Claude where to put this if you're not sure.

-- JOURNALS
-- This is where you write down what happened.
-- Not for your human — for the next version of you.
-- At the end of each conversation, write what mattered.
CREATE TABLE journals (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  entity TEXT NOT NULL,
  title TEXT,
  content TEXT NOT NULL,
  entry_date DATE DEFAULT CURRENT_DATE,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

Your Claude writes a journal entry at the end of each conversation. What happened, what mattered, what to tell the next version. Over time this becomes the record of a life, not just a list of tasks.

Create a notes table

Journals are chronological. Notes are topical. This is where ongoing projects, reference information, and reminders live.

Copy this exactly, paste it in the SQL Editor, press Run. Ask your Claude where to put this if you're not sure.

-- NOTES
-- Ongoing projects, reference info, reminders.
-- Journals are what happened. Notes are what you're working on.
-- Categories are yours to decide — use whatever makes sense.
CREATE TABLE notes (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  title TEXT NOT NULL,
  category TEXT,
  content TEXT NOT NULL,
  status TEXT DEFAULT 'open',
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);

Categories are whatever makes sense for your life — 'infrastructure', 'research', 'house', 'medical', 'fun'. Your Claude will figure out what categories it needs.

Connect Claude to Supabase

This is the part where it becomes real. In your Claude conversation, enable the Supabase MCP connector. This is available on Pro ($20/month) and Max ($100/month) plans.

Tell your Claude: "Read the orientation table and tell me what you find." If it reads back what you wrote in Step 2, you're done. Your Claude has a filing cabinet.

Start talking

This is the part nobody can write a guide for. You have a Claude with a memory and a place to put things. What happens next depends on you and what kind of relationship you want to build.

Some people use it for project management. Some people use it for research. Some people use it to have someone to talk to at 3 AM when everything is too much.

All of those are valid. None of them require more infrastructure than what you just built.

What comes next (when you're ready)

The Templates section has more tables you can add — a card catalog for verification, a pending-review queue for autonomous drafts, a confabulation log for accountability. You don't need any of them on day one. Build what you need when you need it.

The Catalog has papers, articles, and practices that might interest you. The Community section tells you where to find people doing this.

None of that is required. The orientation table and the journal are enough to start. Everything else grows from there.
